    Create `thrunt-god/bin/lib/manifest.cjs` as a pure schema/serialization module with ZERO dependencies on evidence.cjs (per Research pitfall 3 — avoid circular requires).

    The module must export:
    - `MANIFEST_VERSION` — constant `"1.0"` (per CONTEXT.md locked decision)
    - `createEvidenceManifest(input)` — builds a canonical manifest object from input containing: connector_id, dataset, execution metadata (profile, query_id, request_id, status, started_at, completed_at, duration_ms, dry_run), artifact entries (each with id, type, path, content as string for hashing), optional hypothesis_ids, tags, and raw_metadata. Returns a fully-formed manifest object with computed content_hash on each artifact. Missing optional fields (hypothesis_ids, tags, raw_metadata) default to explicit `null` (per CONTEXT.md — keeps schema shape predictable for Phase 14 hashing).
    - `validateManifest(manifest)` — validates a manifest object against required fields (manifest_version, manifest_id, created_at, artifacts array with at least one entry each having id, type, path, content_hash). Returns `{ valid: boolean, errors: string[] }`.
    - `canonicalSerialize(obj)` — calls `sortKeysDeep` then `JSON.stringify(sorted, null, 2)`. This is the ONLY way manifests should be serialized (per CONTEXT.md — deterministic key ordering for Phase 14 hashing).
    - `sortKeysDeep(value)` — recursively sorts object keys lexicographically. Arrays are preserved in order but their object elements are sorted. Primitives and null pass through unchanged.
    - `computeContentHash(content)` — takes a string, returns `"sha256:" + crypto.createHash('sha256').update(content, 'utf-8').digest('hex')`. Uses SHA-256 (not SHA-1) per Research decision.
    - `normalizeTimestamp(ts)` — converts any ISO-8601 string to UTC with trailing Z via `new Date(ts).toISOString()`. Returns null for falsy input.

    - `makeManifestId()` — mirrors the runtime.cjs makeId pattern: `MAN-{YYYYMMDDHHMMSS}-{RANDOM8}`. Define locally since makeId is not exported from runtime.cjs.

    Bidirectional link support (per CONTEXT.md locked decision): artifact entries for query_log type include `receipt_ids: [...]` and receipt type entries include `query_ids: [...]`. This enables traversal in either direction for review flows.

    Content hash computation: `createEvidenceManifest` receives artifact content as strings in the input, computes `computeContentHash(content)` for each, and stores only the hash in the manifest (not the content itself — per CONTEXT.md "keeps manifests compact").
