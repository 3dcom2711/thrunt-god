## Accomplishments
- Hunt Overview verdict cards, phase rail, and activity feed all support Arrow/Home/End key navigation via useRovingTabindex
- Evidence Board graph nodes are keyboard-navigable with ARIA labels describing node type and verdict; Enter key activates same as click
- Evidence Board matrix column headers are keyboard-activatable with tabIndex + role=button
- Query Analysis sort pills, comparison template rows, and receipt inspector list all support roving tabindex
- All navigable groups have proper ARIA roles (role=list/listitem or role=group) and descriptive aria-labels
- Heatmap and matrix tables have aria-label attributes for screen reader context

## Decisions Made
- SVG `<g>` elements receive `tabindex={isDimmed ? -1 : 0}` so dimmed (non-connected) nodes are excluded from keyboard navigation
- Enter key on focused graph nodes triggers the same handleNodeClick action as mouse click
- Matrix column headers get `tabIndex={0}` + `role="button"` + Enter keydown handler for keyboard activation
- Heatmap table relies on native table semantics for screen reader navigation (no roving tabindex)
