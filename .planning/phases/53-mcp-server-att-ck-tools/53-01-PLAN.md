    ```javascript
    'use strict';

    const GROUPS_DATA = path.join(__dirname, '..', '..', 'thrunt-god', 'data', 'mitre-attack-groups.json');
    ```

    **IMPORTANT: Make INTEL_DB_DIR and INTEL_DB_PATH overridable for testing.** Export a `openIntelDb(opts)` that accepts `{ dbPath }` to allow tests to use a temp directory instead of real `~/.thrunt/`. Pattern:
    ```javascript
    function openIntelDb(opts = {}) {
      const dbDir = opts.dbDir || INTEL_DB_DIR;
      const dbPath = opts.dbPath || path.join(dbDir, 'intel.db');
      fs.mkdirSync(dbDir, { recursive: true });
      const db = new Database(dbPath);
      db.pragma('journal_mode = WAL');
      db.pragma('busy_timeout = 5000');
      db.pragma('foreign_keys = ON');
      ensureIntelSchema(db);
      populateIfEmpty(db);
      return db;
    }
    ```

    **Schema** (ensureIntelSchema function):
    ```sql
    CREATE TABLE IF NOT EXISTS techniques (
      id TEXT PRIMARY KEY,
      name TEXT NOT NULL,
      description TEXT,
      tactics TEXT,
      platforms TEXT,
      data_sources TEXT,
      url TEXT
    );
    CREATE VIRTUAL TABLE IF NOT EXISTS techniques_fts USING fts5(
      name, description, id,
      tokenize='porter unicode61'
    );
    CREATE TABLE IF NOT EXISTS groups (
      id TEXT PRIMARY KEY,
      name TEXT NOT NULL,
      aliases TEXT,
      description TEXT,
      url TEXT
    );
    CREATE TABLE IF NOT EXISTS group_techniques (
      group_id TEXT NOT NULL REFERENCES groups(id),
      technique_id TEXT NOT NULL,
      PRIMARY KEY (group_id, technique_id)
    );
    CREATE TABLE IF NOT EXISTS group_software (
      group_id TEXT NOT NULL,
      software_id TEXT NOT NULL,
      PRIMARY KEY (group_id, software_id)
    );
    CREATE TABLE IF NOT EXISTS software (
      id TEXT PRIMARY KEY,
      name TEXT NOT NULL,
      type TEXT,
      description TEXT
    );
    CREATE TABLE IF NOT EXISTS software_techniques (
      software_id TEXT NOT NULL REFERENCES software(id),
      technique_id TEXT NOT NULL,
      PRIMARY KEY (software_id, technique_id)
    );
    CREATE INDEX IF NOT EXISTS idx_gt_tid ON group_techniques(technique_id);
    CREATE INDEX IF NOT EXISTS idx_st_tid ON software_techniques(technique_id);
    ```

    NOTE per RESEARCH.md Pitfall 3: Use regular FTS5 (NOT external content) for techniques_fts since intel.db data is write-once/immutable. This avoids sync issues. The FTS table stores its own copy of text. The FTS5 table includes `id` as a column so searches can be joined back to the techniques table via `id`.

    **Population** (populateIfEmpty function):
    - Check `SELECT COUNT(*) FROM techniques` inside a `BEGIN IMMEDIATE` transaction
    - If count > 0, skip (already populated)
    - Load techniques from TECHNIQUES_DATA JSON
    - For each parent technique: INSERT into techniques (id, name, description, tactics=technique.tactic, platforms=JSON array joined by comma, data_sources=joined by comma, url=`https://attack.mitre.org/techniques/${id.replace('.','/')}/`)
    - For each sub-technique: INSERT into techniques with id=sub.id, name=sub.name, description=parent.description (sub-techniques share parent description in this data format), tactics=parent.tactic, platforms=parent.platforms joined, data_sources=parent.data_sources joined, url derived from sub ID
    - Also INSERT into techniques_fts (name, description, id) for each technique row
    - Load groups from GROUPS_DATA JSON
    - For each group: INSERT into groups table, then INSERT each technique_id into group_techniques, then INSERT each software_id into group_software
    - For each software: INSERT into software table, then INSERT each technique_id into software_techniques
    - COMMIT transaction

    **Query functions** (all exported):
    - `lookupTechnique(db, id)` -- `SELECT * FROM techniques WHERE id = ? COLLATE NOCASE` with id.toUpperCase(). Returns row object or null. If technique has parent (contains '.'), also query parent for additional context.
    - `searchTechniques(db, query, opts)` -- FTS5 MATCH on techniques_fts with BM25 ranking. `opts` accepts `{ tactic, platform, limit }`. SQL: `SELECT t.* FROM techniques t WHERE t.id IN (SELECT id FROM techniques_fts WHERE techniques_fts MATCH ?) [AND t.tactics LIKE ?] [AND t.platforms LIKE ?] ORDER BY t.id LIMIT ?`. For BM25 ranking use: `SELECT id, rank FROM techniques_fts WHERE techniques_fts MATCH ? ORDER BY rank` to get ranked IDs, then look up full rows from techniques table. Recommended implementation: `SELECT t.* FROM techniques t INNER JOIN (SELECT id, rank FROM techniques_fts WHERE techniques_fts MATCH ? ORDER BY rank) AS fts ON t.id = fts.id [WHERE t.tactics LIKE ? AND t.platforms LIKE ?] LIMIT ?`.
    - `lookupGroup(db, id)` -- `SELECT * FROM groups WHERE id = ? COLLATE NOCASE`
    - `getGroupTechniques(db, groupId)` -- `SELECT technique_id FROM group_techniques WHERE group_id = ?`
    - `getGroupSoftware(db, groupId)` -- `SELECT s.* FROM software s JOIN group_software gs ON s.id = gs.software_id WHERE gs.group_id = ?`
    - `getTechniquesByTactic(db, tactic)` -- `SELECT * FROM techniques WHERE tactics LIKE ?` with `%${tactic}%`
    - `getAllTactics(db)` -- `SELECT DISTINCT tactics FROM techniques`, split each by comma, collect unique, sort
