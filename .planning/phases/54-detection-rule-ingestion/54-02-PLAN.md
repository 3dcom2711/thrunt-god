    **Step 2: Wire ensureDetectionsSchema into openIntelDb**

    Edit `mcp-hunt-intel/lib/intel.cjs`:

    1. At the top of the file (after the existing requires), add a lazy require for detections:
    ```javascript
    // Lazy-load detections module (avoids circular dependency; only needed at DB open time)
    let _detections;
    function getDetections() {
      if (!_detections) _detections = require('./detections.cjs');
      return _detections;
    }
    ```

    2. In `openIntelDb()`, after the `ensureIntelSchema(db);` line and before `populateIfEmpty(db);`, add:
    ```javascript
    // Extend schema with detections tables (Phase 54)
    getDetections().ensureDetectionsSchema(db);
    ```

    3. After `populateIfEmpty(db);`, add:
    ```javascript
    // Lazy-populate detections from bundled rules + env var paths (Phase 54)
    getDetections().populateDetectionsIfEmpty(db);
    ```

    This means every call to `openIntelDb()` now creates the detections table (idempotent) and populates it on first run (skipped if already populated). The tools.cjs try/catch blocks around `SELECT ... FROM detections` will now find the table and return real data.

    Do NOT modify the exports of intel.cjs -- the change is internal to openIntelDb only.
