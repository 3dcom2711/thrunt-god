# THRUNT God VS Code Extension -- Data Models & API Specification

This document defines the complete data model, artifact parsers, state management, webview
message protocol, file-system mapping, and cross-reference index for the THRUNT God VS Code
extension. Every interface field is traceable to a specific source file in the
`thrunt-god/bin/lib/` runtime or the `.hunt/` artifact tree.

---

## 1. Domain Model (TypeScript Interfaces)

### 1.1 Identifiers

All IDs follow the `{PREFIX}-{YYYYMMDDHHMMSS}-{RANDOM8}` pattern generated by
`makeId()` in `connector-sdk.cjs:95-98`.


### 3.2 State Derivation

```
File change detected
    |
    v
[FileWatcher] ──> identifies artifact type from path
    |
    v
[ArtifactParser] ──> produces ParseResult<T>
    |
    v
[HuntStore.update()] ──> replaces typed artifact in store
    |
    v
[DerivedIndexer] ──> recomputes affected indexes:
    |                   - queryIndex (if Query changed)
    |                   - receiptIndex (if Receipt changed)
    |                   - hypothesisIndex (if Hypothesis changed)
    |                   - templateIndex (if Query with templates changed)
    |                   - entityIndex (if any artifact with entities changed)
    |
    v
[GraphBuilder] ──> rebuilds EvidenceGraph from all cross-references
    |
    v
[TimelineBuilder] ──> rebuilds timeline from entity timelines in queries
    |
    v
[MessageBus] ──> sends typed update messages to webview
```

## 6. Cross-Reference Index

### 6.1 Relationship Map

The extension builds a cross-reference graph by scanning frontmatter and content
of all artifacts. Here is how each reference type is resolved:

| From Artifact | Relationship | To Artifact | Resolution |
|---|---|---|---|
| Receipt frontmatter `related_queries` | Receipt -> Query | QRY-*.md | Match `query_id` frontmatter field |
| Receipt frontmatter `related_hypotheses` | Receipt -> Hypothesis | HYPOTHESES.md | Match `HYP-NN` in hypotheses list |
| Receipt frontmatter `claim_status` | Receipt -> Hypothesis (edge kind) | HYPOTHESES.md | `supports` / `disproves` / `context` / `inconclusive` |
| Query frontmatter `related_hypotheses` | Query -> Hypothesis | HYPOTHESES.md | Match `HYP-NN` in hypotheses list |
| Query frontmatter `related_receipts` | Query -> Receipt | RCT-*.md | Match `receipt_id` frontmatter field |
| Query body Template Clustering JSON | Query -> Template | In-memory template index | `template_id` from `ReduceEventsOutput.clusters` |
| Query directory path | Query -> Phase | HUNTMAP.md | Phase number inferred from plan references in HUNTMAP |
| Receipt directory path | Receipt -> Phase | HUNTMAP.md | Phase number inferred from plan references in HUNTMAP |
| Receipt body Anomaly Framing | Receipt -> Entity | Entity index | Entity ID parsed from `### Entity:` header |
| FINDINGS.md verdict table `Evidence` column | Findings -> Receipt + Query | QRY/RCT files | Parse RCT-*, QRY-* references from evidence text |
| FINDINGS.md `## What We Know` bullet items | Findings -> Receipt | RCT-*.md | Parse `(RCT-*)` references inline |
| EVIDENCE_REVIEW.md quality checks | EvidenceReview -> Receipt | RCT-*.md | Parse `RCT-*` references in Notes column |
| EVIDENCE_REVIEW.md template clustering table | EvidenceReview -> Query | QRY-*.md | `queryId` in Query column |
| Pack `extends` field | Pack -> Pack | Other pack JSON | Match `id` field in pack registry |
| Pack `expected_progressions` | Pack -> Technique | MITRE ATT&CK | `technique` field in progression steps |
| Receipt Anomaly Framing `Pack progression match` | Receipt -> Pack progression | Pack JSON | Match `progressionName` and step number |
| Manifest `artifacts[].id` | Manifest -> Query / Receipt | QRY/RCT files | Match artifact IDs |
| Manifest `artifacts[].content_hash` | Manifest -> File integrity | Disk file | Recompute SHA-256 of file content |
| Query frontmatter `manifest_id` | Query -> Manifest | MAN-*.json | Match `manifest_id` field |
| Receipt frontmatter `manifest_id` | Receipt -> Manifest | MAN-*.json | Match `manifest_id` field |

### 6.2 Graph Construction Algorithm

```
1. SCAN all .hunt/ files, parse frontmatter for each

2. BUILD node list:
   - One node per hypothesis (from HYPOTHESES.md)
   - One node per query (from QUERIES/QRY-*.md)
   - One node per receipt (from RECEIPTS/RCT-*.md)
   - One node per unique entity (from query entity timelines)
   - One node per Drain template (from query template clustering)
   - One node per phase (from HUNTMAP.md)
   - One node per pack (from loaded pack definitions)

3. BUILD edge list from frontmatter cross-references:
   For each Receipt r:
     For each q in r.related_queries:
       add edge(q -> r, kind='produces', weight=1.0)
     For each h in r.related_hypotheses:
       add edge(r -> h, kind=mapClaimStatus(r.claim_status), weight=1.0)

   For each Query q:
     For each h in q.related_hypotheses:
       add edge(q -> h, kind='queries_for', weight=1.0)
     For each template t in q.templateClustering.clusters:
       add edge(q -> t.template_id, kind='contains_template', weight=0.5)

4. BUILD inferred edges from content analysis:
   For each Receipt r with anomaly framing:
     Extract entity from framing
     add edge(r -> entity, kind='mentions_entity', weight=0.7)
     If pack progression match exists:
       add edge(r -> pack_progression, kind='tested_by', weight=0.8)

   For each Findings verdict:
     Parse receipt/query references from evidence column
     add edge(finding -> referenced_artifact, weight=0.6)

5. PHASE assignment:
   For each Query/Receipt:
     Match query/receipt IDs against plan descriptions in HUNTMAP.md
     add edge(artifact -> phase, kind='in_phase', weight=0.3)

6. PACK hierarchy:
   For each Pack with extends:
     add edge(child_pack -> parent_pack, kind='extends_pack', weight=0.5)
```

### 6.3 Index Maintenance

When an artifact changes, only affected indexes are rebuilt:

| Changed File | Indexes Rebuilt |
|---|---|
| `HYPOTHESES.md` | `hypothesisIndex`, all edges referencing hypotheses |
| `QRY-*.md` | `queryIndex`, `templateIndex` (for that query), entity nodes, edges from that query |
| `RCT-*.md` | `receiptIndex`, edges from that receipt, anomaly frame edges |
| `HUNTMAP.md` | Phase nodes, `in_phase` edges for all queries/receipts |
| `FINDINGS.md` | Finding node, edges to receipts/queries mentioned in verdicts |
| `EVIDENCE_REVIEW.md` | Review node, edges to receipts/queries mentioned in checks |
| `MAN-*.json` | Manifest node, integrity status of linked artifacts |
| `packs/**/*.json` | Pack nodes, `extends_pack` and `tested_by` edges |
